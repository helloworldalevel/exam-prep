  • Portability: Ease of moving a device (Tablets > Desktop PCs).

🔄 1.1.3 Technological Convergence

  • Concept: Multiple separate technologies combining into a single multipurpose device.
  • Example: A smartphone converges a telephone, digital camera, GPS navigator, and portable music player.
  • Impact: Reduces the need to carry multiple devices, lowers overall cost, but drains battery faster.

⚙️ 1.1.4 Embedded Systems

  • Concept: A computer system with a dedicated function within a larger mechanical/electrical system.
  • Features: Low power consumption, small size, rugged operating ranges, low cost per unit.
  • Examples: Washing machine microcontrollers, engine management systems in cars, microwave timers.
💡 Embedded systems usually run a single program continuously and cannot be easily upgraded by the end user like a general-purpose PC.

Topic 1: Hardware & Software

🔌 1.1.5 Firmware

  • Concept: Software programmed permanently into a hardware device, providing low-level control for its specific hardware.
  • Storage: Stored in Non-Volatile Memory (ROM or Flash Memory) so it is not lost when power is removed.
  • Example: A PC's BIOS/UEFI, or the software inside a TV remote control.
💡 Firmware acts as the crucial bridge between the physical hardware and the operating system. Without it, the OS wouldn't know how to talk to the motherboard.

  • Capacity: RAM allows more apps open simultaneously; Storage holds more files.

  • Power Efficiency: Crucial for mobile devices to extend battery life, and data centres to reduce cooling costs.

⏱️ 1.1.7 Data File Size & Transmission Time

  • Formula: Time (seconds) = File Size (bits) / Transmission Speed (bits per second)
  • Crucial Step: Always convert to matching units! Speeds are usually in bits (Mbps), file sizes in bytes (MB).
  • Conversion: 1 Byte = 8 bits.
💡 Show all your working out! If you make a math error at the end but showed the correct conversion to bits, you will still pick up method marks.

Topic 1: Hardware & Software

🔢 1.1.8 Binary & Denary (IEC Standards)

  • Units: Bit (1/0), Nibble (4 bits), Byte (8 bits).
  • IEC Base-2 prefixes:
    • KiB (Kibibyte) = 1024 Bytes
    • MiB (Mebibyte) = 1024 KiB
    • GiB (Gibibyte) = 1024 MiB
  • Conversion: 1011 = (1×8) + (0×4) + (1×2) + (1×1) = 11 in Denary.
💡 IEC standards strictly use 1024 (Base-2), whereas standard SI prefixes (KB, MB) use 1000 (Base-10). Read the question carefully to see which is requested!

📝 Exam Question 3

A file is 10 MiB in size. A network has a bandwidth of 5 Mbps.
Calculate the time it will take to transmit the file. Show your working. [3 marks]

Model Answer:

1. Convert MiB to Bytes: 10 * 1024 * 1024 = 10,485,760 Bytes (1)
2. Convert to bits: 10,485,760 * 8 = 83,886,080 bits (1)
3. Divide by speed: 83,886,080 / 5,000,000 (5 Mbps) = 16.78 seconds (1).

💡 Watch the prefixes! "MiB" means multiply by 1024. "Mbps" (Mega bits per second) usually uses Base-10 (1,000,000) for network speeds.

Topic 2: Networks

🕸️ 2.1.1 Network Models

  • Client-Server: Central server manages resources, security, and backups. Highly secure but expensive to set up.
  • Peer-to-Peer (P2P): All PCs have equal status. Cheaper, but security/backups are left to individual users.
  • Ad hoc networks: Temporary networks created directly between devices without a router (e.g., Apple AirDrop).
  • Tethering: Sharing a mobile device's cellular data connection with other devices (acting as a hotspot).
💡 P2P is great for a small home network (sharing a printer), but Client-Server is essential for schools/businesses for central data security.
Slide 22
Topic 2: Networks

📻 2.1.2 Network Communication Protocols

  • Wi-Fi: Wireless LAN protocol using radio waves. High bandwidth, medium range.
  • Bluetooth: Short-range wireless for peripherals (headphones, mice).
  • ZigBee: Low-power, low-data rate wireless protocol used for smart home automation (IoT devices like smart bulbs).
  • Cellular (3G/4G/5G): Wide Area Network over cell towers.
  • Ethernet: Standard wired LAN protocol using copper cables.
💡 ZigBee is preferred over Wi-Fi for smart home sensors because it uses significantly less battery power and forms a mesh network.
Slide 23
Topic 2: Networks

📞 2.1.3 Network Standards & Protocols

  • TCP/IP: The foundational suite of rules for the Internet. TCP ensures reliable packet delivery; IP routes the packets.
  • VoIP (Voice Over IP): Transmitting voice calls over internet broadband rather than analog phone lines (e.g., Skype, Teams).
  • SIP (Session Initiation Protocol): Used to set up, maintain, and terminate VoIP calls.
  • 7-Layer OSI Model: Conceptual framework standardising network communication across different hardware manufacturers.
💡 Know the difference: VoIP is the actual transmission of the voice data, while SIP is the protocol that "rings" the other person and establishes the connection.
Slide 24
Topic 2: Networks

🔌 2.2.1 Network Transmission Media (Wired)

  • Copper (Twisted Pair): Common Ethernet cable. Cheap, flexible, but susceptible to electromagnetic interference (EMI).
  • Fibre Optic: Uses pulses of light through glass. Extremely fast, very high bandwidth, long-distance, no EMI. Expensive and fragile.
  • Powerline: Uses existing electrical wiring in a building to transmit data. Great for extending a network without drilling walls.
💡 If an exam scenario involves connecting two separate buildings on a campus, always recommend Fibre Optic for the "backbone" due to its speed and resistance to weather/EMI.
Slide 25
Topic 2: Networks

📡 2.2.1 Network Transmission Media (Wireless)

  • Radio: Covers Wi-Fi/Bluetooth. Passes through walls but prone to interference from other devices.
  • Microwave: Point-to-point communication. Requires direct line-of-sight between dishes.
  • Infrared (Light): Requires direct line-of-sight, cannot penetrate solid objects (e.g., TV remote).
  • Satellite: Relays signals from space. Excellent for remote areas, but suffers from high latency (delay).
💡 Don't confuse Microwave and Satellite. Microwave is terrestrial (tower to tower on Earth), Satellite bounces off orbit. Both require line-of-sight!
Slide 26
Topic 2: Networks

📊 2.2.2 Network Metrics (Part 1)

  • Bandwidth: Theoretical maximum data capacity of a network connection (the width of the pipe).
  • Throughput: The actual, real-world rate of successful data delivery (always lower than bandwidth).
  • Latency: The time delay for a data packet to travel from source to destination.
  • Jitter: The variation in latency over time. Highly disruptive to real-time apps.
💡 Students often use "Speed" incorrectly. Use "Bandwidth" for capacity, "Latency" for delay, and "Jitter" for variation in arrival times.
Slide 27
Topic 2: Networks

📉 2.2.2 Network Metrics (Part 2)

  • Error Rate: Measure of the number of bits that are transferred incorrectly (corrupted data).
  • Packet Loss: Packets that completely fail to reach their destination. Causes buffering.
  • Scalability: How easily the network can expand without performance drops.
  • Availability: The percentage of time a network is operational (uptime).
💡 Do not confuse "Error Rate" (corrupted bits arriving) with "Packet Loss" (data never arriving at all).
Slide 28
Topic 2: Networks

🖲️ 2.2.3 Network Components (Part 1)

  • Switch: Connects devices on a LAN. Forwards data *only* to the specific MAC address intended (intelligent). Reduces collisions.
  • Hub: Legacy device. Broadcasts data to all ports regardless of destination. High collision rate.
  • Router: Connects different networks together (e.g., LAN to Internet) using IP addresses.
  • Gateway: Connects two entirely different networks using different protocols (translates the data).
💡 Switches use MAC addresses (Layer 2). Routers use IP addresses (Layer 3). Know the difference!
Slide 29
Topic 2: Networks

🔌 2.2.3 Network Components (Part 2)

  • WAP (Wireless Access Point): Allows wireless devices to connect to a wired network via Wi-Fi.
  • NIC (Network Interface Card): Internal hardware allowing a device to connect to a network. Contains the burned-in MAC address.
  • Repeater: Boosts/regenerates a signal to extend its physical range.
  • Server: High-performance PC providing network services (File, Print, Web) to clients.
💡 A standard home "Wi-Fi Router" is actually a Multi-function device: it contains a Router, a Switch, and a WAP all in one box!

Topic 2: Networks

🌍 2.2.5 IP Addressing

  • IPv4: 32-bit address (e.g., 192.168.1.1). Running out of unique addresses globally.
  • IPv6: 128-bit address. Solves address exhaustion.
  • Static IP: Manually assigned, never changes. Essential for servers and printers so they can always be found.
  • Dynamic IP: Temporarily assigned when a device connects. Good for mobile phones.
  • DHCP: The server protocol that automatically hands out Dynamic IP addresses.
💡 Logical (IP) addresses are used to route data across the internet. Physical (MAC) addresses route data locally within a LAN.

Topic 2: Networks

🛡️ 2.3.1 Network Security Issues

  • Threats: Malware (Viruses, Trojans, Ransomware), Phishing, DoS/DDoS attacks, unauthorized access (hacking).
  • Impacts: Data loss, financial theft, severe reputation damage, legal fines (GDPR).
  • Open Networks: Public Wi-Fi transmits data unencrypted by default. Hackers can easily intercept data using "packet sniffers" (Man-in-the-Middle attack).
💡 Never just say "hacking is bad". Specify *why*: "A data breach could result in heavy GDPR fines and loss of customer trust."
Slide 33
Topic 2: Networks

🧱 2.3.2 Securing Networks (Firewalls & Anti-malware)

  • Firewall: Hardware/software that monitors incoming/outgoing traffic based on security rules. Blocks unauthorized access.
  • Anti-malware:
    • Signature-based: Uses a database of known malware code. Must be updated daily.
    • Heuristic detection: Monitors running processes for suspicious behavior (e.g., a program trying to encrypt all files). Catches new, unknown viruses.
💡 Students struggled to name "Heuristic detection". Use this exact term when asked how anti-malware catches brand-new viruses without a database!
Slide 34
Topic 2: Networks

🔐 2.3.2 Securing Networks (Encryption Types)

  • Symmetric Encryption: The sender and receiver use the SAME shared key to encrypt and decrypt the message. Fast, but hard to share the key securely.
  • Asymmetric Encryption: Uses a mathematically linked pair of keys. The sender encrypts with a Public Key. The receiver decrypts with their Private Key. Highly secure.
  • End-to-End Encryption: Ensures only the communicating users can read the messages (e.g., WhatsApp).
💡 Learn these specific terms! Don't just guess "Client-Server" or "Public Encryption" when asked about encryption techniques.
Slide 35
Topic 2: Networks

👤 2.3.2 Securing Networks (Authentication & Physical)

  • Authentication: Verifying identity.
    • Passwords: Knowledge factor (weakest if simple).
    • Two-Factor (2FA): Password + SMS code / App.
    • Biometrics: Physical traits (fingerprint, iris). Hardest to fake.
  • Access Rights: Restricting what users can see (e.g., Student vs Teacher folders).
  • Physical Controls: Server room locks, CCTV, disabling unused Ethernet wall ports.
💡 Biometrics are highly secure but raise privacy issues. Discuss how storing children's fingerprints in a school database could be a target for identity theft.

Topic 3: Online Environment

🌐 3.1.1 Internet Structure

  • The Internet: The physical, global network of interconnected computer networks using TCP/IP.
  • IP Addressing: Every device needs a unique IP (like a postal address) to send/receive data across this global network.
  • DNS (Domain Name System): The internet's phonebook. Translates human-readable domain names (www.pearson.com) into computer-readable IP addresses (192.168.1.1).
💡 The Internet is the hardware (cables, routers). The World Wide Web is the software/service (webpages) that runs ON the internet.
Slide 42
Topic 3: Online Environment

🕸️ 3.1.2 & 3.1.3 World Wide Web & Content Types

  • Features: Uses HTTP/HTTPS protocols, accessed via browsers, relies on URLs and HTML.
  • Static Web Pages: Written in plain HTML/CSS. Content looks exactly the same to every user. Fast to load, cheap to host. Good for a basic read-only contact page.
  • Dynamic Web Pages: Content changes based on user interaction, time, or location by querying a database. Slower to load, requires backend servers (e.g., an Amazon shopping cart or Facebook feed).
💡 If an exam asks for a suitable webpage for an online store, ALWAYS choose dynamic, as the site must update stock levels and handle individual user carts.
Slide 43
Topic 3: Online Environment

📜 3.1.4 Client-side Scripting

  • Concept: Code (like JavaScript) runs locally inside the user's web browser, NOT on the web server.
  • Role: Validates web forms (e.g., checking a password is 8 chars long before submitting), creates interactive UI elements (animations).
  • Pros: Immediate feedback to the user, reduces processing load on the web server.
  • Cons: Code is visible to the user (insecure); user can disable scripts in their browser.
💡 Never use client-side scripting for security (like checking a password against a database). A smart user can bypass client-side code!
Slide 44
Topic 3: Online Environment

⚙️ 3.1.5 Server-side Scripting

  • Concept: Code (like PHP, Python) executed on the web server before the final HTML is sent to the user's browser.
  • Role: Interacting with databases, processing secure logins, generating dynamic HTML pages based on user requests.
  • Pros: Secure (user cannot see the source code), reliable (doesn't depend on user browser settings).
  • Cons: Increases processing load on the server, requires an internet round-trip for every action.
💡 Combining both is best: Client-side for fast UI validation, Server-side for secure database interaction.

  • Cloud Storage: Storing data on remote third-party servers accessed via the internet.

  • Concept: Breaking a complex, large IT system down into smaller, manageable sub-systems.
  • Why? Makes the system easier to design, build, test, and debug. Different developer teams can work on different sub-systems simultaneously.

  • Concept: Visualizes how data moves through a system. Focuses purely on data, NOT control logic, decisions, or time.

🔀 4.3.1 & 4.3.2 Flowcharts

  • Concept: Shows the step-by-step sequence of operations or algorithms. Shows control flow (logic), unlike DFDs.
  • Symbols:
    • (Oval) Terminator: Start / End.
    • [Rectangle] Process: Action/Calculation (e.g., Calculate total).
    • /Parallelogram/ Input/Output: Reading/Writing data (e.g., Display error).

💡 Students are generally good at flowchart notation. Ensure every Diamond has TWO labelled exit arrows (Yes/No)!

📊 5.1.1 Data vs Information

  • Data: Raw, unprocessed facts, figures, and symbols with no context or meaning.
    Example: 10052024, 45.
  • Information: Data that has been processed, structured, or given context to make it meaningful and useful for decision making.

Topic 5: Data & Databases

📁 5.1.2 Data Sources & Types

  • Structured Data: Highly organised data that fits neatly into a predefined format, like rows and columns in a relational database (Customer Names, Ages, Postcodes). Easy to search and query via SQL.
  • Unstructured Data: Data with no predefined format. Harder to search and analyze. (Videos, Audio files, Free-text emails, Social media posts).
💡 Machine Learning/AI is increasingly used to extract meaningful information from massive amounts of unstructured data (like reading customer reviews to determine sentiment).
Slide 71
Topic 5: Data & Databases

🗄️ 5.2.1 Why Databases?

  • Spreadsheets are poor for huge datasets and complex relationships.
  • Databases are used to:
    • Structure data to reduce redundancy (duplication).
    • Enforce data integrity (accuracy and consistency).
    • Allow multiple users to query and update data concurrently without conflicts.
    • Provide granular security (access rights).
💡 "Redundancy" is the magic word. Typing a customer's address 50 times in a spreadsheet causes errors. A database stores it once in a Customer table.
Slide 72
Topic 5: Data & Databases

🔑 5.2.2 Relational Database Structure

  • Tables (Entities): Hold data about one specific theme (e.g., Student Table).
  • Records (Rows): All the data relating to a single instance (one student).
  • Fields (Columns): A single piece of data (Date of Birth).
  • Primary Key (PK): A unique identifier for EVERY record (e.g., StudentID).
  • Foreign Key (FK): A field in one table that links to the Primary Key in another table, creating the relationship.
💡 A Composite Key is a primary key made of TWO OR MORE fields combined to create a unique identifier (e.g., Date + EmployeeID).
Slide 73
Topic 5: Data & Databases

🔗 5.2.3 Entities and Relationships

  • One-to-One (1:1): Rare. E.g., One Country has One Capital City.
  • One-to-Many (1:M): Most common. E.g., One Mother can have Many Children. One Customer places Many Orders.
  • Many-to-Many (M:M): E.g., Many Students can take Many Subjects. Cannot be implemented directly in a relational DB. MUST be resolved using a junction/link table.
💡 ERD Notation (Crow's Foot): A straight line ending in a dash = 'One'. A line splitting into three prongs = 'Many'.
Slide 74
Topic 5: Data & Databases

💻 5.3.1 Structured Query Language (SQL)

  • Purpose: The standard language used to interact with relational databases.
  • Data Manipulation Language (DML): Used to add, modify, delete, and retrieve specific records (SELECT, INSERT, UPDATE).
  • Data Definition Language (DDL): Used to build the database itself (CREATE TABLE, Define Data Types like VARCHAR, INT).
💡 SQL is universally understood by different database systems (MySQL, Oracle, SQL Server), making it a vital standard.
Slide 75
Topic 5: Data & Databases

📝 5.3.2 Basic SQL Commands

  • SELECT: Queries data.
    SELECT * FROM Students WHERE Age > 16;
  • INSERT: Adds records.
    INSERT INTO Students (Name) VALUES ('John');
  • UPDATE: Modifies data.
    UPDATE Students SET Age = 17 WHERE Name = 'John';
  • DELETE: Removes data.
    DELETE FROM Students WHERE Name = 'John';
💡 The asterisk * means "Select ALL fields". Don't forget the semicolon ; at the end of statements!
Slide 76
Topic 5: Data & Databases

🔍 5.3.2 SQL Features (Wildcards & Aggregation)

  • Wildcards: Used with LIKE for pattern matching.
    % represents zero or more characters (LIKE 'A%' finds all starting with A).
  • Aggregate Functions: COUNT(), SUM(), AVG(), MIN(), MAX().
  • ORDER BY: Sorts results alphabetically or numerically (ASC/DESC).
💡 Do NOT use LIKE when comparing dates or exact numbers. Use >= instead! E.g., WHERE published >= "01/01/1965".
Slide 77
Topic 5: Data & Databases

🔗 5.3.2 SQL Linking Tables

  • INNER JOIN: Combines columns from two tables based on a related column (PK to FK). Returns rows where there is a match in BOTH tables.
    SELECT Customers.Name, Orders.Date
    FROM Customers
    INNER JOIN Orders ON Customers.CustID = Orders.CustID;
  • UNION: Combines the result sets of two SELECT statements into a single column vertically. Tables must have the same number of columns/data types.
💡 If asked to return a set of unique titles available in ONE OR BOTH tables, use UNION.
SELECT title FROM tbl_book UNION SELECT title FROM tbl_dvd;

🧠 Command Words: The Basics

  • Identify / State / Give: Requires a brief, direct answer. Usually 1 mark. No explanation or linking needed. Just hit the keyword.
  • Describe: Paint a picture in words. Give characteristics/features of a system. Tell the examiner what it is or how a process works step-by-step.
💡 If a question asks you to "Describe the process...", list the steps in chronological order. 1 mark per correct step.

Exam Strategy

🗣️ Command Words: The Explain Question

  • Explain: Give a statement, then say why or how.
  • The Golden Rule: You must provide a linked expansion.
  • Use connecting phrases: "Because...", "This means that...", "Which leads to...".
  • Usually 2-4 marks. 1 mark for the point, 1 mark for the expansion.
💡 "Responses to a two-mark explain must be linked. Where responses gave two statements, they could only be awarded one mark."

⚖️ Command Words: The Essays

  • Discuss: Explore different aspects of an issue. Give reasons for and against. Look for nuances.
  • Evaluate: The highest mark questions (6 to 12 marks). Requires weighing up points.
  • Structure: You MUST provide Advantages, Disadvantages, AND a justified Conclusion. Do not just sit on the fence; make a decision based on the scenario!
💡 "The length of the essay is not necessarily related to the awarded mark... Long, rambling essays with vague info get no more marks than short, accurate ones."
